       object EsajSelectors {
           // URL paths
           const val SHOW_PATH = "/cpopg/show.do"
           const val SEARCH_PATH = "/cpopg/search.do"

           // Process header fields (ASSUMED — verify in live spike)
           const val CLASSE = "#classeProcesso"
           const val ASSUNTO = "#assuntoProcesso"
           const val FORO = "#foroProcesso"
           const val VARA = "#varaProcesso"
           const val JUIZ = "#juizProcesso"

           // Parties table

           // Incidents

           // Search page
           const val SEARCH_RESULT_TABLE = "#listagemDeProcessos"

       }

    2. Create `src/main/kotlin/br/com/precatorios/scraper/EsajScraper.kt` as `@Service`:
       - Constructor inject: `ScraperProperties`, `esajRateLimiter: RateLimiter`, `esajRetry: Retry` (qualified by bean name)
       - Define data classes inside or as top-level in same file:
         - `data class ProcessoScraped(val numero: String, val classe: String?, val assunto: String?, val foro: String?, val vara: String?, val juiz: String?, val valorAcao: String?, val partes: List<ParteScraped>, val incidentes: List<IncidenteScraped>, val missingFields: List<String>, val rawHtml: String)`
         - `data class ParteScraped(val nome: String, val tipo: String?, val advogado: String?)`
         - `data class IncidenteScraped(val numero: String?, val descricao: String?, val link: String?)`
         - `data class BuscaResultado(val numero: String, val classe: String?, val assunto: String?, val foro: String?)`

       - `fun fetchProcesso(numero: String): ProcessoScraped`:
         Wrap with programmatic `RateLimiter.decorateCheckedSupplier(esajRateLimiter) { Retry.decorateCheckedSupplier(esajRetry) { doFetchProcesso(numero) }.apply() }.apply()`
         Inside `doFetchProcesso`:
         - Build URL: `"${properties.esaj.baseUrl}${EsajSelectors.SHOW_PATH}"`
         - Jsoup.connect(url).data("processo.codigo", numero).userAgent(properties.esaj.userAgent).timeout(properties.esaj.timeoutMs).get()
         - Check response status: if 429, throw `TooManyRequestsException("e-SAJ HTTP 429 for processo $numero")`
         - Extract fields using `doc.select(EsajSelectors.CLASSE).firstOrNull()?.text()` pattern
         - For each nullable field, if null, log warning `log.warn("Selector '{}' returned no match for processo {}", selectorName, numero)` and add field name to `missingFields` list (per D-04, D-05, D-06)
         - Extract parties: `doc.select(EsajSelectors.PARTES_TABLE)` then iterate rows to build `List<ParteScraped>`
         - Extract incidents: `doc.select(EsajSelectors.INCIDENTES_TABLE)` then iterate rows/links to build `List<IncidenteScraped>`
         - Store raw HTML in `rawHtml` field for `dados_brutos` JSONB storage
         - Return `ProcessoScraped` with all extracted data

       - `fun buscarPorNome(nome: String): List<BuscaResultado>`:
         Wrap with same Resilience4j pattern.
         Inside `doBuscarPorNome`:
         - Build URL: `"${properties.esaj.baseUrl}${EsajSelectors.SEARCH_PATH}"`
         - Jsoup.connect(url).data("dadosConsulta.pesquisaLivre", nome).userAgent(...).timeout(...).get()
         - Parse result table from `doc.select(EsajSelectors.SEARCH_RESULT_TABLE)`
         - Return `List<BuscaResultado>`

       IMPORTANT: Do NOT throw on missing selectors (per D-04). Return partial data with missingFields populated.
